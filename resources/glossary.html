GenAI Glossary

Essential terminology for generative AI development

About This Glossary

This glossary provides definitions for key terms and concepts in the field of generative AI. Whether you're new to the field or need a quick reference, these definitions will help you understand the terminology used throughout the training materials.

A

Alignment

The process of training an AI system to behave according to human values, intentions, and expectations. Includes techniques like RLHF and Constitutional AI.

Attention Mechanism

A neural network component that enables models to focus on specific parts of input sequences when generating outputs. The core innovation behind transformer architecture.

Agent

An AI system capable of autonomous or semi-autonomous action in an environment, often involving planning, decision-making, and tool use to accomplish tasks.

Auto-regressive Model

A model that generates outputs sequentially, with each new token conditioned on previously generated tokens. GPT models are examples of auto-regressive language models.

C

Context Window

The maximum number of tokens a language model can process at once, including both the input prompt and the generated output. Determines how much information the model can consider when generating responses.

Chunking

The process of breaking down documents or text into smaller segments for more effective processing, embedding, and retrieval in systems like RAG.

Constitutional AI

An approach to AI alignment where models are guided by explicit principles or rules (a "constitution") and trained to critique and improve their own outputs according to these principles.

E

Embedding

A numerical vector representation of text, images, or other data that captures semantic meaning and enables mathematical operations like similarity comparisons.

Embeddings Model

A specialized model designed to convert text or other inputs into vector embeddings that capture semantic relationships for use in search, clustering, or as inputs to other models.

F

Few-shot Learning

The ability of a model to perform a task with only a few examples provided in the prompt, rather than requiring extensive fine-tuning.

Fine-tuning

The process of further training a pre-trained model on a specific dataset or for a specific task to improve its performance in that domain.

H

Hallucination

When an AI model generates content that is incorrect, misleading, or not grounded in provided context or real-world facts.

L

LangChain

A framework for developing applications with LLMs through composable components for working with language models, prompts, memory, and more.

LangGraph

A framework built on LangChain that uses a graph-based approach to create stateful, multi-step workflows for language model applications.

LLM (Large Language Model)

A type of AI model trained on vast amounts of text data to understand and generate human-like text. Examples include GPT-4, Claude, and Llama.

LoRA (Low-Rank Adaptation)

A parameter-efficient fine-tuning technique that significantly reduces the number of trainable parameters by focusing on low-rank updates to pre-trained weights.

M

MCP (Model Context Protocol)

A standardized protocol for connecting language models to external tools, systems, and data sources, allowing models to perform actions beyond their training.

Multimodal AI

AI systems capable of processing and generating content across multiple types of data, such as text, images, audio, and video.

Mixture of Experts (MoE)

A neural network architecture that routes inputs to different specialized sub-networks ("experts") based on the input type, enabling larger and more efficient models.

P

Parameter

A value in a neural network that is learned during training. Large language models typically have billions to trillions of parameters that encode learned patterns from training data.

Prompt Engineering

The practice of designing and optimizing input prompts to effectively communicate with and control the output of language models.

Pre-training

The initial training phase of an LLM on a large corpus of text data, where it learns general language understanding before any task-specific fine-tuning.

Q

Quantization

A technique that reduces model size and inference time by representing weights with lower precision (e.g., converting 32-bit floating point numbers to 8-bit or 4-bit integers).

R

RAG (Retrieval-Augmented Generation)

An approach that enhances language model outputs by retrieving relevant information from external knowledge sources and incorporating it into the context provided to the model.

RLHF (Reinforcement Learning from Human Feedback)

A technique for training language models using human preferences, where a reward model is trained on human comparisons of outputs and then used to optimize the language model.

ReAct (Reasoning + Acting)

A pattern for agent behavior that interleaves reasoning steps with actions, allowing models to plan, act, and reflect in an iterative process.

T

Temperature

A parameter that controls the randomness of language model outputs. Higher values (e.g., 0.8-1.0) produce more creative, diverse responses, while lower values (e.g., 0.2-0.4) yield more deterministic, focused outputs.

Tokenization

The process of converting text into smaller units (tokens) that can be processed by a language model. Tokens may be words, subwords, or individual characters.

Top-k Sampling

A text generation strategy that limits token selection to the k most probable next tokens, filtering out less likely options.

Top-p (Nucleus) Sampling

A text generation strategy that selects from the smallest set of tokens whose cumulative probability exceeds the threshold p, providing a dynamic cutoff based on probability distribution.

Transformer

A neural network architecture introduced in 2017 that relies on self-attention mechanisms to process sequential data in parallel, forming the foundation of modern language models.

Tool Calling

The ability of an LLM to generate structured calls to external tools or APIs, enabling it to perform actions outside its training data.

V

Vector Database

A specialized database designed to store and efficiently query vector embeddings, enabling semantic search and similarity matching for RAG and other applications.

Z

Zero-shot Learning

The ability of a model to perform tasks it wasn't explicitly trained on, without any examples provided in the prompt.
